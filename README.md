# Telegram-бот для задач руководителя и ассистента

MVP на Python, python-telegram-bot и SQLite.

## Как пользоваться

После `/start` бот показывает кнопки внизу Telegram.

Руководитель видит кнопку:

- `Создать задачу`

Ассистент видит:

- `Сдать результат`
- `Список задач`
- `Ждут руководителя`
- `Сводка`
- `Мой Telegram ID`

Если один человек тестирует обе роли, его Telegram ID можно указать сразу в `ASSISTANT_IDS` и `MANAGER_IDS`.

## Сценарий

1. Руководитель нажимает `Создать задачу`.
2. Бот просит написать задачу в формате:

```text
Название | дедлайн | комментарий
```

3. Ассистент получает задачу и выбирает статус кнопками:

- `В работе`
- `Ждёт руководителя`
- `Готово на проверку`
- `Нужны вводные`
- `Зависло`
- `Выполнено`

4. После выбора статуса ассистент пишет комментарий руководителю.
5. Когда задача готова, ассистент нажимает `Сдать результат`.
6. Результат можно отправить текстом, фото или файлом.
7. Руководитель получает результат и выбирает:

- `Одобрить`
- `Нужны правки`
- `Задать вопрос`
- `Позже`

Если руководитель выбирает правки, вопрос или позже, бот просит написать комментарий для ассистента.

## Команды всё ещё работают

- `/start`
- `/whoami`
- `/new Название | дедлайн | комментарий`
- `/submit ID | результат`
- `/list`
- `/waiting`
- `/summary`
- `/done ID`

## Файлы

- `bot.py` - Telegram-бот.
- `database.py` - SQLite-база задач.
- `config.py` - настройки из `.env`.
- `cli_test.py` - локальная проверка без Telegram.
- `.env.example` - пример настроек без настоящего токена.
- `requirements.txt` - библиотеки Python.
- `tests/test_smoke.py` - базовый тест сценария.

## Настройки окружения

На Bothost добавьте переменные:

```env
BOT_TOKEN=токен_от_BotFather
ASSISTANT_IDS=telegram_id_ассистента
MANAGER_IDS=telegram_id_руководителя
DATABASE_PATH=assistant_tasks.db
REMINDER_AFTER_MINUTES=60
DIGEST_HOUR=9
DIGEST_MINUTE=0
```

Настоящий `.env` нельзя загружать в GitHub.

## Запуск локально

```powershell
py -m pip install -r requirements.txt
py bot.py
```
