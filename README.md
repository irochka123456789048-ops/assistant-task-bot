# Telegram-бот для задач руководителя и ассистента

MVP на Python, python-telegram-bot и SQLite.

## Как пользоваться

После `/start` бот показывает кнопки внизу Telegram.

Руководитель видит кнопку:

- `Создать задачу`

Ассистент видит:

- `Сдать результат`
- `Список задач`
- `Ждут руководителя`
- `Сводка`
- `Мой Telegram ID`

Если один человек тестирует обе роли, его Telegram ID можно указать сразу в `ASSISTANT_IDS` и `MANAGER_IDS`.

## Сценарий

1. Руководитель нажимает `Создать задачу`.
2. Бот просит написать задачу обычным текстом.

```text
Подготовить договор
```

Если нужен дедлайн или комментарий, можно написать расширенно:

```text
Подготовить договор | завтра 18:00 | проверить сумму
```

Дедлайн и комментарий необязательны.

3. Ассистент получает задачу и выбирает статус кнопками:

- `В работе`
- `Жду решения`
- `На согласовании`
- `Жду комментарии`
- `Зависло`
- `Перенос`
- `Отмена`
- `Выполнено`

4. После выбора статуса ассистент пишет комментарий руководителю.
Руководитель получает обновление и кнопку `Дать комментарий`, чтобы ответить ассистенту по любому статусу.
5. Когда задача готова, ассистент нажимает `Сдать результат`.
6. Бот показывает список задач. Ассистент выбирает нужную задачу кнопкой.
7. Результат можно отправить текстом, фото или файлом.
7. Руководитель получает результат и выбирает:

- `Одобрить`
- `Нужны правки`
- `Задать вопрос`
- `Позже`

Если руководитель выбирает правки, вопрос или позже, бот просит написать комментарий для ассистента.

Кнопка `Сводка` показывает статусы как раскрываемый список: можно нажать на статус, увидеть задачи внутри, затем открыть карточку конкретной задачи и посмотреть детали.

Кнопка `Ждут руководителя` показывает задачи со статусами: `Жду комментарии`, `На согласовании`, `Жду решения`, `Зависло`, `Перенос`.

## Задачи из Telegram-группы

Можно добавить бота в рабочую группу. Тогда руководитель сможет ставить задачи прямо там.

Если группа используется только для задач, руководитель просто пишет задачу обычным сообщением:

```text
Подготовить договор
```

Или расширенно:

```text
Подготовить договор | пятница | проверить сумму
```

Бот создаёт задачу из любого текстового сообщения руководителя в группе. Сообщения от людей, которых нет в `MANAGER_IDS`, игнорируются.

Чтобы бот видел сообщения в группе:

1. Добавьте бота в группу.
2. Сделайте бота администратором группы.
3. В `@BotFather` откройте вашего бота: `Bot Settings` -> `Group Privacy` -> `Turn off`.

## Файлы

- `bot.py` - Telegram-бот.
- `database.py` - SQLite-база задач.
- `config.py` - настройки из `.env`.
- `cli_test.py` - локальная проверка без Telegram.
- `.env.example` - пример настроек без настоящего токена.
- `requirements.txt` - библиотеки Python.
- `tests/test_smoke.py` - базовый тест сценария.

## Настройки окружения

На Bothost добавьте переменные:

```env
BOT_TOKEN=токен_от_BotFather
OPENAI_API_KEY=ключ_OpenAI_для_расшифровки_голосовых
ASSISTANT_IDS=telegram_id_ассистента
MANAGER_IDS=telegram_id_руководителя
DATABASE_PATH=assistant_tasks.db
REMINDER_AFTER_MINUTES=60
DIGEST_HOUR=9
DIGEST_MINUTE=0
```

Настоящий `.env` нельзя загружать в GitHub.

## Запуск локально

```powershell
py -m pip install -r requirements.txt
py bot.py
```

## Локальная проверка без Telegram

```powershell
py cli_test.py
```

## Голосовые задачи из группы

Если руководитель отправит голосовое сообщение в рабочую группу, бот расшифрует его и создаст задачу из текста.

Для этого нужно:

1. Добавить бота в группу.
2. Сделать бота администратором.
3. В `@BotFather` выключить `Group Privacy`.
4. На Bothost добавить переменную `OPENAI_API_KEY`.
