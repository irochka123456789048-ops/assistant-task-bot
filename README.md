# Telegram-бот для задач руководителя и ассистента

MVP на Python, python-telegram-bot и SQLite.

## Новый рабочий сценарий

1. Руководитель создаёт задачу:

```text
/new Подготовить договор | завтра 18:00 | проверить сумму
```

2. Ассистенту приходит задача с кнопками статусов:

- `В работе`
- `Ждёт руководителя`
- `Готово на проверку`
- `Нужны вводные`
- `Зависло`
- `Выполнено`

3. Ассистент выбирает статус и пишет комментарий руководителю.

Например:

```text
Приняла в работу. Нужен шаблон прошлого договора.
```

4. Руководитель получает обновление статуса и комментарий ассистента.

5. Когда задача выполнена, ассистент сдаёт результат:

```text
/submit 1 | Договор готов, сумму и реквизиты проверила
```

Можно сдать файл или фото: прикрепите документ/фото и в подписи напишите:

```text
/submit 1 | Готовый файл во вложении
```

6. Руководитель получает результат и выбирает:

- `Одобрить`
- `Нужны правки`
- `Задать вопрос`
- `Позже`

Если руководитель выбирает правки, вопрос или позже, бот просит написать комментарий. Ассистент получает обратную связь.

## Команды

- `/start` - подсказка.
- `/whoami` - показать ваш Telegram ID.
- `/new Название | дедлайн | комментарий` - руководитель создаёт задачу.
- `/submit ID | результат` - ассистент сдаёт результат.
- `/list` - активные задачи.
- `/waiting` - задачи, которые ждут руководителя.
- `/summary` - сводка по статусам.
- `/done ID` - вручную отметить выполненной.

## Файлы

- `bot.py` - Telegram-бот.
- `database.py` - SQLite-база задач.
- `config.py` - настройки из `.env`.
- `cli_test.py` - локальная проверка без Telegram.
- `.env.example` - пример настроек без настоящего токена.
- `requirements.txt` - библиотеки Python.
- `tests/test_smoke.py` - базовый тест сценария.

## Настройка `.env`

Создайте `.env` на основе `.env.example`:

```powershell
Copy-Item .env.example .env
```

Заполните:

```env
BOT_TOKEN=токен_от_BotFather
ASSISTANT_IDS=telegram_id_ассистента
MANAGER_IDS=telegram_id_руководителя
DATABASE_PATH=assistant_tasks.db
REMINDER_AFTER_MINUTES=60
DIGEST_HOUR=9
DIGEST_MINUTE=0
```

Настоящий `.env` нельзя загружать в GitHub. Он уже добавлен в `.gitignore`.

## Запуск

```powershell
py -m pip install -r requirements.txt
py bot.py
```

## Локальная проверка без Telegram

Если Telegram API недоступен, проверьте сценарий через консоль:

```powershell
py cli_test.py
```

В меню можно:

- создать задачу от руководителя;
- выбрать статус от ассистента и написать комментарий;
- сдать результат текстом или условным файлом;
- выбрать решение руководителя и написать комментарий;
- посмотреть список задач и summary.
